Types of Programming Languages

Topic 02 · How Humans Talk to Computers

Learning Objectives

  • Explain the difference between low-level and high-level languages.
  • Describe compiled languages versus interpreted languages.
  • Identify examples of popular programming languages and their common uses.
  • Explain where Python fits among other programming languages.
Getting Started

What is a Programming Language?

Computers only understand electrical signals. Programming languages let humans give them clear, structured instructions.

  • Definition: A programming language is a set of rules and words used to write instructions that a computer can follow.
  • Like a real language: It has its own vocabulary (keywords) and grammar (syntax) that must be followed exactly.
  • Simple explanation: If a computer only understands 0s and 1s, a programming language is the translator that turns human ideas into those 0s and 1s.
TipEvery app, website, and game you've ever used was written in one or more programming languages.

Programming languages are how humans and computers "talk" to each other.

Concept

Low-Level Languages

Low-level languages speak in a way that is very close to what the computer's hardware understands.

  • Machine Language: Pure binary code (0s and 1s). This is the only language the CPU truly executes.
  • Assembly Language: Uses short human-readable codes (like MOV, ADD) that map directly to machine instructions.
  • Very fast and efficient because they are so close to the hardware — but hard for humans to read and write.
machine-vs-assembly.txt 
Machine Language:   10110000 01100001
Assembly Language:   MOV AL, 61h
NoteBeginners rarely write low-level code directly — but it's good to know it exists underneath every program.
Concept

High-Level Languages

High-level languages read much closer to plain English, making them easier for humans to learn and write.

  • Readable syntax: Words like if, print, and for are used instead of raw binary codes.
  • Portable: The same code can often run on different types of computers with little or no change.
  • Examples: Python, Java, C++, JavaScript, and many more.

print("Hello!") reads almost like plain English.

ExampleWriting print("Hello!") in Python is far easier than writing the equivalent instructions in raw machine code.
Comparison

Low-Level vs. High-Level

Each type trades off readability for speed, or speed for readability.

AspectLow-Level LanguageHigh-Level Language
ReadabilityHard for humans to readEasy for humans to read
SpeedVery fastSlightly slower
PortabilityTied to specific hardwareRuns on many machines
ExamplesMachine code, AssemblyPython, Java, C++
TipAs a beginner, you will almost always start with a high-level language — that's exactly why this course uses Python.
Concept

Compiled vs. Interpreted Languages

High-level code must be translated into machine code before (or while) it runs. There are two ways this happens.

Source Code
Compiler / Interpreter
Machine Code
  • Compiled: The entire program is translated into machine code first, producing a separate executable file, then run.
  • Interpreted: Code is translated and run line-by-line, on the spot, with no separate executable file.
Translation Method

Compiled Languages

A compiler translates the whole program at once, before it ever runs.

  • Code is checked for errors all at once, before execution begins.
  • Produces a standalone executable file (like program.exe) that can run without the original source code.
  • Tends to run very fast once compiled.
  • Examples: C, C++, Rust.

Compiling is like translating an entire book before handing it to the reader.

Translation Method

Interpreted Languages

An interpreter translates and runs code one line at a time, as the program executes.

  • No separate executable file — you need the interpreter installed to run the code.
  • Easier to test and debug, since you can run code immediately without a separate compile step.
  • Usually a little slower than compiled code, since translation happens while running.
  • Examples: Python, JavaScript.

Interpreting is like a live translator repeating each sentence as it's spoken.

NotePython is an interpreted language — that's part of why it's so beginner-friendly and quick to test.

Why It Matters

Why This Course Uses Python

Python sits in a sweet spot for beginners: simple to read, yet powerful enough for real jobs.

  • Readable syntax: Python code often looks like plain English instructions.
  • Huge community: If you get stuck, thousands of tutorials and forums can help.
  • Versatile: Used in web development, data science, artificial intelligence, and automation.
  • Fast to write: You can build small, working programs in just a few lines of code.
ExamplePrinting "Hello, World!" takes about 5 lines in some languages, but only 1 line in Python.
Applied Example

"Hello, World!" in Different Languages

The classic first program every programmer writes — notice how much shorter the Python version is.

hello_world.py 
print("Hello, World!")
HelloWorld.java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
TipFewer lines doesn't always mean "better" — but for beginners, less to type means fewer chances to make mistakes.
Recap

Summary — Key Points

Here's everything we covered in this topic, in one glance.

Low-level languages are close to hardware; high-level languages are closer to human language.
Compiled languages translate all at once; interpreted languages translate line-by-line.
Different languages exist for different jobs: Python, Java, C, C++, JavaScript, and more.
Python is high-level and interpreted, which makes it great for beginners.

Did You Know?

Fun Facts About Programming Languages

A few fun tidbits to impress your friends.

Named After Comedy

Python was named after the British comedy show "Monty Python's Flying Circus," not the snake!

Thousands Exist

There are thousands of programming languages, but only a few dozen are widely used today.

Old but Gold

Some languages like C (created in 1972) are still widely used today, over 50 years later.

Scratch for Kids

Scratch, a drag-and-drop visual language, is often the very first programming language kids learn.
