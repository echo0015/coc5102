String Access & Slicing

Topic 05 · Reaching Into a String for Exactly What You Need

Learning Objectives

  • Access individual characters in a string using indexing.
  • Use negative indexing to count from the end of a string.
  • Extract substrings using slicing, including a step value.
  • Explain why strings are immutable in Python.
Getting Started

Strings Are Sequences of Characters

A string isn't just one block of text — it's an ordered sequence, one character at a time.

  • Definition: Every character in a string has a fixed position, called an index.
  • Simple explanation: Think of a string as a row of labeled boxes, each holding one character.

"PYTHON" is 6 characters, each with its own position.

Syntax

Indexing Basics

Use square brackets to access a single character by its position, starting from 0.

indexing.py 
word = "PYTHON"
print(word[0])   # P
print(word[3])   # H
ImportantPython counts starting at 0, not 1 — the first character is always index 0.

Negative Indexing

Negative indexes count backward from the end of the string.

negative_index.py 
word = "PYTHON"
print(word[-1])   # N  (last character)
print(word[-2])   # O  (second to last)
Tipword[-1] is the easiest way to grab the last character, no matter how long the string is.
Tool

Finding a String's Length

The len() function counts exactly how many characters are in a string.

string_length.py 
word = "PYTHON"
print(len(word))       # 6
print(word[len(word) - 1])   # N (last character, the long way)
NoteThe last valid positive index is always len(word) - 1, since counting starts at 0.
Syntax

Slicing Basics

Slicing extracts a whole substring, using word[start:stop].

slicing_basics.py 
word = "PYTHON"
print(word[0:2])   # PY
print(word[2:5])   # THO
ImportantThe slice includes start but stops before stop — word[0:2] gives only 2 characters, not 3.
Syntax

Slicing with a Step

A third number, word[start:stop:step], controls how many characters to skip each time.

slicing_step.py 
word = "PYTHON"
print(word[0:6:2])   # PTO  (every 2nd character)
ExampleStep 2 means "take a character, then skip one" — starting at P, then T, then O.
Shortcut

Omitting Start, Stop, or Step

Leaving a part blank tells Python to use a sensible default.

SliceMeaningResult on "PYTHON"
word[:3]From the start up to index 3PYT
word[3:]From index 3 to the endHON
word[:]The whole string (a copy)PYTHON
word[::-1]The whole string, reversedNOHTYP
Tipword[::-1] is the most popular one-line trick to reverse a string in Python.
Concept

Strings Are Immutable

Once a string is created, its individual characters can never be changed in place.

immutable_error.py 
word = "PYTHON"
word[0] = "J"

# TypeError: 'str' object does not support item assignment
ImportantTo "change" a string, you must build a brand-new one, e.g. "J" + word[1:].
Reference

A few slicing tricks come up again and again in real programs.

Applied Example

Example: Extracting a Username from an Email

Slicing is perfect for pulling out just the part of a string you actually need.

extract_username.py 
email = "jerico@thelewiscollege.edu.ph"
at_position = email.find("@")

username = email[:at_position]
print(username)   # jerico
Notefind("@") locates the position of "@", then slicing grabs everything before it.
Recap

Summary — Key Points

Here's everything we covered in this topic, in one glance.

word[i] accesses one character; indexing starts at 0.
Negative indexes count backward from the end.
word[start:stop:step] slices out a substring.
Strings are immutable — slicing creates new strings, never edits in place.

Did You Know?

Fun Facts About Strings

A few fun tidbits to impress your friends.

Slicing Never Errors on Range

Unlike single indexing, slicing outside the string's length just returns less text instead of crashing.

word[:] Makes a Copy

Slicing the entire string with no start or stop creates a full copy, which is a handy trick in more advanced code.

Zero-Based Counting

Starting at 0 instead of 1 is called "zero-based indexing" — nearly every modern language does this.

Lists Work the Same Way

Everything you learned here — indexing, negative indexing, slicing — works identically on Python lists too.
