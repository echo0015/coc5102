Flowchart Fundamentals

Topic 06 · Planning Logic Before Writing Code

Learning Objectives

  • Identify standard flowchart symbols and their meaning.
  • Draw a simple flowchart for an everyday process.
  • Translate a flowchart into a logical sequence of steps.
  • Explain why planning with flowcharts helps avoid coding errors.
Getting Started

What is a Flowchart?

A flowchart is a picture that shows the steps of a process, in order, using simple shapes and arrows.

  • Definition: A flowchart is a visual diagram that represents the steps and decisions in a process or program.
  • Simple explanation: It's like a map for your logic — instead of streets, you follow arrows from step to step.

Flowcharts turn abstract logic into something you can see.

Why It Matters

Why Use Flowcharts?

Planning before coding saves time and helps you catch mistakes early.

  • Plan first: You can design the logic of a program before writing a single line of code.
  • Communicate clearly: Flowcharts help explain your logic to teammates, teachers, or clients who may not read code.
  • Spot errors early: Gaps or missing steps in logic are much easier to see in a diagram than in lines of code.
  • Language-independent: A flowchart's logic can later be written in Python, Java, or any other language.
Reference

Standard Flowchart Symbols

Each shape in a flowchart has one specific meaning — this consistency is what makes flowcharts easy to read.

SymbolShapeMeaning
TerminalOvalMarks the Start or End of the flowchart
ProcessRectangleA single action or instruction to perform
DecisionDiamondA yes/no question that branches the flow
Input/OutputParallelogramGetting input from, or showing output to, the user
Flow LineArrowShows the direction the process moves next
Symbol

The Terminal Symbol (Start / End)

Every flowchart begins with a Start oval and finishes with an End oval.

Start
Steps...
End
NoteA flowchart should only ever have one Start symbol, though it can sometimes have more than one End.
Symbol

The Process Symbol

A rectangle represents one clear action or calculation the program performs.

  • Used for actions like calculations, assignments, or any single step.
  • Each rectangle should describe just one clear action.

Example: "Add 5 to total"

Symbol

The Decision Symbol

A diamond asks a yes/no question and splits the flow into two different paths.

Age ≥ 18?
Yes → Allow Entry
Age ≥ 18?
No → Deny Entry
ExampleA decision diamond always has exactly two paths leading out: one for "Yes" and one for "No."
Symbol

The Input/Output Symbol

A parallelogram shows where data enters the program, or where results are shown to the user.

  • Input: "Ask the user for their name" or "Read a number from the keyboard."
  • Output: "Display the total" or "Show the result on screen."
TipIf a step is getting information in or showing information out, it belongs in a parallelogram, not a rectangle.

Example: Making a Cup of Coffee

A simple, everyday process with no decisions — just steps in order.

Applied Example

Example: Is a Number Even or Odd?

This time, a decision diamond splits the flow into two possible outcomes.

  • Start
  • Input: Read a number
  • Decision: Is number % 2 == 0?
  • Yes path → Output: "It's Even"
  • No path → Output: "It's Odd"
  • End
NoteNotice both the "Yes" and "No" paths eventually lead to the same End symbol.
Concept

From Flowchart to Code

Once your logic is planned in a flowchart, turning it into Python code becomes much easier.

  • Input/Output symbols become input() and print().
  • Decision diamonds become if statements.
  • Process rectangles become regular lines of code.
even_odd.py 
number = int(input("Enter a number: "))

if number % 2 == 0:
    print("It's Even")
else:
    print("It's Odd")

Did You Know?

Fun Facts About Flowcharts

A few fun tidbits to impress your friends.

Older Than Computers

Flowcharts were first used in industrial engineering in the 1920s, long before modern computers existed.

ISO Standard

Flowchart symbols are standardized by ISO 5807, so they mean the same thing worldwide.

Pseudocode Cousin

Flowcharts and pseudocode both describe logic before real code is written — just in different formats.

Used Everywhere

Flowcharts aren't just for programmers — businesses use them to map workflows and decision-making too.
