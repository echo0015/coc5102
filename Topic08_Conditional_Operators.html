Conditional Operators

Topic 08 · Comparing Values to Build Conditions

Getting Started

What Are Conditional Operators?

Conditional (or relational) operators compare two values and always produce True or False.

  • Definition: A comparison asks a yes/no question about two values, like "is A bigger than B?"
  • Always boolean: The result is always exactly True or False — never anything else.

Comparisons are the building blocks of every decision your program makes.

Reference

The Conditional Operators

These six operators cover every possible way to compare two values.

Closer Look

Comparing Numbers and Strings

Comparison operators work on more than just numbers — they also work on strings.

compare_types.py 
print(5 == 5.0)          # True  (same value, different type is OK)
print("cat" == "cat")    # True  (identical text)
print("cat" == "Cat")    # False (case matters!)
NoteString comparisons are case-sensitive — "Cat" and "cat" are treated as completely different values.
Neat Trick

Chained Comparisons

Python lets you check if a value falls within a range using one compact line.

chained.py 
age = 20
print(1 < age < 100)       # True — same as: 1 < age and age < 100
ExampleMost other programming languages need two separate comparisons joined together — Python lets you chain them directly.
Applied Example

Example: Comparing Two Scores

Comparisons are how programs decide which value "wins."

compare_scores.py 
player1 = 85
player2 = 92

print(player1 > player2)   # False
print(player2 > player1)   # True
print(player1 == player2)  # False
Watch Out

Common Mistake: = vs. ==

This is the single most common beginner mistake in conditions.

wrong_vs_right.py 
# Wrong — this assigns, it doesn't compare, and causes an error here
if score = 90:
    print("Perfect!")

# Correct
if score == 90:
    print("Perfect!")
ImportantWhenever you're checking a condition, you need two equals signs: ==.

A 7 >= 7

Answer: True — 7 is equal to 7, and >= includes equal values.

B "10" == 10

Answer: False — "10" is a string and 10 is an integer; different types are never equal.

Answer: False — 3 does equal 3, so "not equal" is False.
Did You Know?

Fun Facts About Conditional Operators

A few fun tidbits to impress your friends.

Chained Comparisons

Python lets you write 1 < x < 10 directly — most other languages require two separate comparisons.

Strings Compare Alphabetically

Python can compare strings like "apple" < "banana" based on alphabetical (dictionary) order.

Booleans Are Numbers

Since True equals 1 and False equals 0 internally, True == 1 actually evaluates to True.

Foundation of Logic

Every if statement, loop condition, and search you'll ever write relies on comparison operators.
