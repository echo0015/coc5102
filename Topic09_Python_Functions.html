Topic 09 · Organizing Code Into Reusable Pieces

Getting Started

What is a Function?

A function is a named, reusable block of code that performs a specific task.

  • Definition: A function groups instructions together under one name, so you can run them anytime just by calling that name.
  • Analogy: Think of a function like a kitchen recipe — you can follow it (call it) as many times as you like without rewriting it.

A function packages instructions into one reusable unit.

Why It Matters

Why Use Functions?

Functions help you write cleaner, more organized, and less repetitive programs.

  • Avoid repetition: Write the logic once, then reuse it as many times as needed.
  • Organize code: Breaking a big program into small functions makes it easier to read and understand.
  • Easier to fix: If something goes wrong, you only need to fix it in one place.
  • Teamwork: Different people can build and test separate functions independently.
Syntax

Defining a Function

The def keyword creates a new function in Python.

define_function.py 
def greet():
    print("Hello there!")
    print("Welcome to Programming 1.")
  • def tells Python you're creating a function.
  • greet is the function's name — choose something clear and descriptive.
  • The indented lines below make up the function's body.
Syntax

Calling a Function

Defining a function doesn't run it — you must call it by name to actually execute its code.

call_function.py 
def greet():
    print("Hello there!")

greet()   # This line actually runs the function
greet()   # You can call it again and again
NoteNotice the parentheses () after the function name — that's what tells Python "run this function now."
Concept

Parameters and Arguments

Functions can accept input values, called parameters, to make them more flexible.

greet_name.py 
def greet(name):
    print("Hello,", name + "!")

greet("Maria")   # Hello, Maria!
greet("Juan")    # Hello, Juan!
  • Parameter: name — the placeholder listed in the function definition.
  • Argument: "Maria" — the actual value you pass in when calling the function.
Concept

Return Values

A function can send a result back to wherever it was called using the return keyword.

return_demo.py 
def add(a, b):
    return a + b

result = add(3, 4)
print(result)   # 7
Tipprint() only displays something — return actually sends a value back so it can be stored or reused.
Concept

Default Parameter Values

You can give a parameter a default value, used automatically if no argument is provided.

default_param.py 
def greet(name="friend"):
    print("Hello,", name + "!")

greet()          # Hello, friend!
greet("Maria")   # Hello, Maria!
ExampleDefault values are handy for optional settings that most callers won't need to change.
Concept

Local vs. Global Scope

A variable's scope determines where in the program it can be used.

ScopeWhere It's CreatedWhere It Can Be Used
LocalInside a functionOnly inside that same function
GlobalOutside any functionAnywhere in the program
NoteA variable created inside a function disappears once the function finishes running.

Applied Example

Example: Calculating Rectangle Area

Let's write a reusable function that calculates the area of any rectangle.

rectangle_area.py 
def calculate_area(width, height):
    return width * height

room_area = calculate_area(4, 5)
garden_area = calculate_area(10, 3)

print("Room area:", room_area)
print("Garden area:", garden_area)
NoteThe same function is reused twice with completely different numbers — that's the real power of functions.

Did You Know?

Fun Facts About Functions

A few fun tidbits to impress your friends.

Everything Is an Object

In Python, functions themselves can be stored in variables and passed around like any other value.

Functions Can Call Themselves

A function that calls itself is called "recursion" — an advanced technique you may explore later.

Docstrings

Python functions can include a special comment called a docstring that explains what the function does.

Building Blocks of Libraries

Every Python library you'll ever use is really just a huge collection of functions someone else wrote.
