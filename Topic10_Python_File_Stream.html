Python File Stream

Topic 10 · Making Your Programs Remember Data

Learning Objectives

  • Open, read, and close a text file in Python.
  • Write and append data to a file.
  • Explain the purpose of file modes (r, w, a).
  • Use the with statement to handle files safely.
Getting Started

What is File Handling?

Variables disappear once a program ends — files let your data survive after the program closes.

  • Definition: File handling means reading data from, or writing data to, files stored on disk.
  • Why it matters: Without files, every variable's value is lost the moment your program stops running.

Files let programs save information permanently.

Syntax

Opening a File

The open() function is the starting point for any file operation in Python.

open_file.py 
file = open("notes.txt", "r")
ModeMeaning
"r"Read — opens a file for reading only (default mode)
"w"Write — creates a new file or erases an existing one
"a"Append — adds new content to the end without erasing what's there
Syntax

Reading a File

Python offers a few different ways to read the contents of a file.

MethodReads
read()The entire file, as one single string
readline()Just one line at a time
readlines()All lines, as a list of strings
read_file.py 
file = open("notes.txt", "r")
content = file.read()
print(content)
file.close()
Syntax

Writing to a File

Opening a file in "w" mode lets you write new content into it.

write_file.py 
file = open("notes.txt", "w")
file.write("Hello, this is my first saved note!")
file.close()
Important"w" mode erases the file's previous contents before writing — use "a" mode instead if you want to keep what's already there.
Syntax

Appending to a File

The "a" mode adds new content to the end of a file, keeping everything that was already there.

append_file.py 
file = open("notes.txt", "a")
file.write("\nThis line was added later.")
file.close()
ExampleA logging system that records new events every day would always use append mode, never write mode.
Best Practice

Closing a File

Always close a file once you're finished with it.

  • Saves changes: Closing ensures anything you wrote is properly saved to disk.
  • Frees resources: An open file can block other programs (or parts of your own program) from using it.
close_file.py 
file = open("notes.txt", "r")
content = file.read()
file.close()   # always remember this step!
Best Practice

The with Statement

The with statement automatically closes a file for you, even if an error happens.

with_statement.py 
with open("notes.txt", "r") as file:
    content = file.read()
    print(content)

# File is automatically closed here!
TipMost professional Python code uses with instead of manually calling close() — it's safer and easy to forget otherwise.
Applied Example

Example: A Simple To-Do List Saver

Let's write a program that saves a task, then reads it back.

todo_saver.py 
with open("tasks.txt", "a") as file:
    file.write("Finish Programming 1 homework\n")

with open("tasks.txt", "r") as file:
    print(file.read())
NoteEach time this program runs, a new task line is appended, and the file keeps growing over time.
Watch Out

Common Error: FileNotFoundError

Trying to read a file that doesn't exist yet causes a common beginner error.

error_example.py 
with open("missing_file.txt", "r") as file:
    print(file.read())

# FileNotFoundError: [Errno 2] No such file or directory
TipReading requires the file to already exist, but writing ("w") or appending ("a") will automatically create it if it's missing.
Recap

Summary — Key Points

Here's everything we covered in this topic, in one glance.

open() opens a file in read ("r"), write ("w"), or append ("a") mode.
read(), readline(), and readlines() all read file content differently.
write() saves new content into a file.
The with statement handles closing the file safely and automatically.

Fun Facts About File Handling

A few fun tidbits to impress your friends.

Beyond .txt Files

Python can also read and write CSV, JSON, and even image files using the same core ideas.

Every App Uses This

Every time you save a document or a game, the software behind it is doing file writing just like this.

Foundation of Databases

Even complex databases are, deep down, built on the same idea of reading and writing files.

Cross-Platform

The same file-handling code works the same way on Windows, macOS, and Linux.
