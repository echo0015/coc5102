Python Errors & Exceptions

Topic 16 · Handling Problems Gracefully with try/except

Learning Objectives

  • Differentiate between syntax errors and runtime errors (exceptions).
  • Identify common types of exceptions in Python.
  • Use try/except to handle errors without crashing the program.
  • Use else, finally, and raise to control exception flow.
Getting Started

What is an Error?

An error is anything that stops your program from running the way you expected.

  • Syntax Errors: Break Python's grammar rules — the program can't even start.
  • Runtime Errors (Exceptions): The code is valid, but something goes wrong while it's running.
TipErrors aren't a sign you're bad at programming — every programmer, beginner or expert, deals with them daily.

An error is Python's way of saying "something's not right here."

Error Type

Syntax Errors

A syntax error means your code breaks Python's grammar rules, so it never even starts running.

syntax_error.py 
if age >= 18
    print("Adult")

# SyntaxError: expected ':'
NoteSyntax errors are usually easy to fix — Python tells you exactly which line looks wrong, before your program ever runs.
Error Type

Runtime Errors (Exceptions)

The code is written correctly, but something unexpected happens while it's actually running.

runtime_error.py 
number = 10
result = number / 0

# ZeroDivisionError: division by zero
ExampleThis code is perfectly valid Python — the problem only shows up once it actually tries to divide by zero.
Reference

Common Exception Types

Python gives each kind of runtime problem its own named exception.

ExceptionHappens When...
ValueErrorA value has the right type but an invalid value, e.g. int("abc")
TypeErrorAn operation is used on the wrong data type, e.g. "2" + 2
ZeroDivisionErrorDividing a number by zero
IndexErrorAccessing a list position that doesn't exist
KeyErrorLooking up a dictionary key that doesn't exist
FileNotFoundErrorOpening a file that doesn't exist
NameErrorUsing a variable that was never defined
Syntax

The try/except Statement

try/except lets your program recover from an error instead of crashing.

basic_try_except.py 
try:
    number = int(input("Enter a number: "))
    print("You entered:", number)
except:
    print("That wasn't a valid number.")
  • Python runs the code inside try first.
  • If an exception happens, Python immediately jumps to the except block instead of crashing.
Best Practice

Catching Specific Exceptions

Naming the exact exception type is safer than catching every possible error the same way.

specific_except.py 
try:
    number = int(input("Enter a number: "))
except ValueError:
    print("Please enter a valid whole number.")
ImportantA bare except: catches everything, even mistakes you didn't expect — that can hide real bugs. Naming the exception keeps you in control.
Concept

Multiple except Blocks

You can handle different kinds of errors differently, in the same try statement.

multiple_except.py 
try:
    divisor = int(input("Enter a divisor: "))
    result = 10 / divisor
    print("Result:", result)
except ValueError:
    print("That's not a number.")
except ZeroDivisionError:
    print("Cannot divide by zero.")
NotePython checks each except in order and runs only the first one that matches the error that occurred.
Syntax

The else Clause

The else block runs only when the try block succeeds with no exception at all.

try_else.py 
try:
    number = int(input("Enter a number: "))
except ValueError:
    print("Invalid input.")
else:
    print("You entered:", number)
ExampleUse else for code that should only run when nothing went wrong — keeping it separate from the risky code in try.
Syntax

The finally Clause

The finally block always runs — whether an exception happened or not.

try_finally.py 
try:
    file = open("data.txt", "r")
except FileNotFoundError:
    print("File not found.")
finally:
    print("Done attempting to open the file.")
Tipfinally is perfect for cleanup steps, like closing a file, that must happen no matter what.
Concept

Raising Exceptions Manually

The raise keyword lets you trigger your own exception when something doesn't make sense.

raise_demo.py 
age = -5

if age < 0:
    raise ValueError("Age cannot be negative.")
NoteRaising your own exceptions helps catch invalid data early, with a clear message about what went wrong.
Applied Example

Example: Validating User Input

Combining a loop with try/except keeps asking until the user gives valid input.

validate_input.py 
while True:
    try:
        age = int(input("Enter your age: "))
        break
    except ValueError:
        print("Please enter a whole number.")

print("Thanks! You are", age, "years old.")
ExampleThe loop only stops (break) once the user finally types something valid — no crash, no confusion.
Recap

Summary — Key Points

Here's everything we covered in this topic, in one glance.

Syntax errors break Python's grammar; exceptions happen while running.
try/except catches errors so your program doesn't crash.
else runs on success; finally always runs for cleanup.
raise lets you trigger your own exception on invalid data.

Did You Know?

Fun Facts About Errors & Exceptions

A few fun tidbits to impress your friends.

Everything is an Exception

In Python, even things like ZeroDivisionError are objects built from a common base class called Exception.

Older Than You Think

Try/except style error handling has existed in programming languages since the 1960s.

Used in Real Software

Every app that shows "Something went wrong, please try again" instead of crashing is using exception handling like this.

Better Safe Than Sorry

Professional Python code almost always wraps file access and user input in try/except, since those are the most error-prone operations.
